Help

In many routine Drosophila crosses set up in research labs, it is important to know if the intended genes are passed down to the progeny. This is done using markers that are added along with the gene, or in trans with it. But in some cases, the same phenotype could arise out of different genotypes. In a usual fly lab scenario, this is unfavourable, as there is no one-isto-one relationship between genoptype and phenotype. This tool is designed to show the user when such phenotype clashes occur.

Another common cause of concern is genetic recombination. Genetic recombination is often undesirable in crosses set up in the lab, and balancer chromosomes are used widely to prevent this. The tool also shows when such recombinations will occur.

In case you have come here for a classical Mendelian genetics teaching tool, you might be disappointed. This tool assumes you know which chromosomes the markers and genes are on, and also some basic drosophila genetics. Thus, even the Punnett square it generates is suited more to the fly researcher than the student. And this is not a replacement for programs of the 'Virtual fly lab' kind. In case that is what you need, look here, or an open source version here. (Source code here), [either of which we're no way related to]

General guidelines

All genotypes follow standard Drosophila genetic nomenclature. ; separates different pairs of chromosomes. / separates chromosomes within a pair. If no / is present, it is assumed to be homozygous. And commas separate genes within a chromosome. And + denotes a wild-type chromosome. An example valid fly would be: +;UAS TntG, UAS GFP/CyO;tub Gal80ts and would be interpreted as +/+ ; (UAS TntG),(UAS GFP)/CyO ; (tub Gal80ts)/(tub Gal80ts)

Input pane

Male

This field is mandatory. Enter the genotype of the male parent, using standard drosophila gene nomenclature. Make sure that there is a Y chromosome in the first pair. And make sure it is an uppercase(capital) Y. Example: +/Y ; OK371 Gal4 ; MKRS

Female

This field is mandatory. Enter the genotype of the female parent, using standard drosophila gene nomenclature. Make sure that there is no Y chromosome in the first pair. Example: +/FM7a ; + ; shi ts

Progeny

This field is optional. Enter the genotype of the F1 progeny you are interested in. Example: Y/FM7a ; OK371 Gal4/+ ; shi ts/MKRS could be the progeny you are interested in.

Balancers

This field is optional. Enter all the balancers used in the cross. Use commas to separate the genes. Example: FM7a,MKRS

Markers

This field is optional. Enter all the markers used in the cross. Use commas to separate the genes. If the first letter of a marker is uppercase(capital), it is treated as a dominant marker, else it is treated as recessive. Example: FM7a,Tb,e where FM7a, Tb are treated as dominant and e is treated as recessive.

Constraints

This field is optional. Constraints are ways of mentioning interactions between genes which affect the cross.

The format is gene1,gene2....geneN:tag for a single constraint. And only one constraint per line. where tag can be one of these:
l - lethal
rl - rescues from lethality
i - marker interference
s - sterile
rs - rescues sterility

l - lethal

Constraint should have atleast one gene. Example: OK371 Gal4, UAS TntG :l or shi:l

rl - rescues from lethality

Constraint should have atleast two genes. It will rescue the lethality caused by any subset of the genes mentioned. Example: OK371 Gal4, UAS TntG , tub Gal80:rl rescues the lethality caused by OK371 Gal4, UAS TntG :l

i - marker interference

Constraint should have atleast two genes. Some markers interfere with each other, and make it phenotypically indistinguishable. For example, Sb and Pin both affect the hair. But screening for a fly containing both of them is a bad idea. Example: Sb,Pin:i

s - sterile

Constraint should have atleast one gene. Some combinations of genes make a fly sterile. When this happens in the progeny, this does not have any consequences unless that fly is intended to be used in another cross. Example: FM7a,FM7a:s

rs - rescues from sterility

Constraint should have atleast two genes. Rescues sterilty caused by anysubset of genes. Example: a,b,c:rs rescues lethality caused by a,c:rs

Output pane

Punnett Square

This panel shows all the possible outcomes from the cross. And highlights the progeny of your choice. The background color of each cell represents a phenotype, and the foreground color represents genotype. A dark grey background colour represents lethality. If you hover the mouse pointer over a cell, other cells with the same phenotype are highlighted. If the highliht is red, it means there is a phenotype clash (i.e. a fly with a different genotype which has the same phenotype as the selected cell). If it is blue, it means there is no phenotype clash with the highlighted cell.

Click on a cell, to select a particular progeny and view its details on the info panel. Click anywhere outside to clear the selection

Info Panel

This panel shows all the details of the selected fly, including recombination warnings, lethality, sterility and phenotype clashes
